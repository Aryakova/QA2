Tест-план для раздела "Корпоративное обучение" (Школа 21)

Объект тестирования:
Раздел «Групповые интенсивы для руководителей и команд»:
https://21-school.ru/corporate-training/gruppovye-intensivy-dlia-rukovoditelei-i-komand
1. Введение

1.1 Цель тестирования

    Проверить функциональность, usability и корректность отображения раздела для корпоративных клиентов.

1.2 Область тестирования

    Верстка и адаптивность.

    Работоспособность форм (заявки, контакты).

    Контент (тексты, изображения, видео).

    Интеграции (если есть: CRM, платежи).

1.3 Вне области тестирования

    Бэкенд-логика (если нет доступа к API).

    Нагрузочное тестирование.

2. Подход и стратегия

2.1 Типы тестирования

    Функциональное: Проверка форм, кнопок, навигации.

    UI/UX: Соответствие макету, читаемость текста.

    Кросс-браузерное: Chrome, Firefox, Safari.

    Мобильное тестирование: iOS/Android.

2.2 Критерии начала/окончания тестирования

    Старт: Готовность тестового окружения.

    Завершение: Исправление всех критических багов.

3. Ресурсы

3.1 Команда

    Тестировщик (я).

    Разработчик (для уточнения багов).

3.2 Инструменты

    Браузеры.

    Программы для баг-трекинга (например Jira).

    Средства для тест-кейсов (например TestRail).
    
4. Расписание

    Планирование.

    Тест-дизайн.

    Выполнение тестов.
    
    Анализ результатов.

5. Тестовые сценарии

5.1 Основные проверки

    Заголовок страницы

        Соответствует ли содержимому (<title>).

    Форма заявки

        Валидация полей (email, телефон).

        Отправка данных (мок API или проверка в CRM).

    Контент

        Нет битых изображений/ссылок.

        Видео загружается.

    Адаптивность

        Верстка на различных устройствах.

        Кроссбраузерная проверка.
        
    Тестирование безопасности

     Проверка защиты формы обратной связи
      
         проверка на XSS-уязвимость
        
         проверка CAPTCHA 
    
     Проверка HTTPS
    
        страница загружается по защищённому протоколу HTTPS

5.2. Пример тест-кейса

ID: CORP-001

Название: Проверка отправки формы заявки.

Шаги:

    Заполнить поля: имя, email, телефон.

    Нажать «Отправить».
    Ожидаемый результат:

    Появление сообщения «Ваша заявка принята».

    Данные сохраняются в системе.

6. Риски

    Нет доступа к реальной CRM → тестируем на заглушке.

    Сроки могут сорваться из-за незапланированных багов.

7. Отчетность

    Ежедневный отчет о прогрессе.

    Финальный отчет с списком багов и статистикой.

8. Приложения

    Макеты дизайна (если есть).

    Чек-листы.
